MANUAL DE USUARIO

 

Una vez arrancada para poder trabajar con el simulador se realizaría lo siguiente:

-NUEVO: Se mostrará en la pantalla tres áreas de texto:

-Esquina Superior Izquierda: Para escribir las instrucciones a ensamblar

-Esquina Superior Derecha: En este caso no se podría escribir nada. Aquí se mostraría el código sin comentarios ni líneas en blanco y ensamblado (Todo ésto si el código al ensamblar no tiene errores).

-Parte Inferior: No editable en ella se mostraran los errores (Si los hubiera).

Al pulsar este botón lo único que se podrá hacer es escribir código (Esquina Superior Izda). Los otros dos paneles aparecen en blanco y no se pueden escribir.


-OPCIONES: Permite cambiar el tiempo que tarda en en ejecutase cada subciclo. Se cambiará el tiempo cuando se pulse Tiempo.


-CÓDIGO: Una vez escrito el código que se desea ensamblar al pulsar el botón EJECUTAR si no hay errores y se pulsa esta opción se mostrarían:

-Esquina Superior Izquierda: El código escrito que no se puede modificar.

-Esquina Superior Derecha: En este caso no se podría escribir nada. Aquí se mostraría el código sin comentarios ni líneas en blanco y ensamblado (Todo ésto si el código al ensamblar no tiene errores).

-Parte Inferior: No editable en ella se mostraran los errores (Si los hubiera).

Si hubiera errores se podría modificar el código escrito.


-EJECUTAR: Al pulsar este botón ensambla el código:

-Si no tiene errores pasaría a la simulación.

-Si tiene errores se dejaría la pantalla que tenemos para poder modificar los errores.

Al volver a pulsarlo se para totalmente la simulación.


-RUTA: Muestra la ejecución de la ruta de datos. Si la simulación no está en ejecución lo que hace es mostrarnos un dibujo de la ruta.


-MEMORIA: Una vez ensamblado el código (sin errores) podemos comprobar en cada momento el estado de la memoria.


-PARAR: Para la simulación del código o la activa dependiendo del estado anterior. Se puede usar siempre que la simulación se esté ejecutando.


-AYUDA: Ayuda para utilizar el programa.
